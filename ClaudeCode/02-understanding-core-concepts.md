# 02. 핵심개념 이해하기

Claude Code는 기본 도구 외에 아래 10가지 확장 개념으로 이루어진다.

각 개념이 에이전트 루프의 어느 지점에 연결되는지 이해하는 것이 핵심이다.

## 1. CLAUDE.md

모든 세션에서 자동으로 읽히는 지속 컨텍스트 파일이다.

- 프로젝트 루트, 사용자 홈, 하위 디렉토리에 계층적으로 둘 수 있다.
- Claude가 항상 알아야 할 내용을 넣는다.
- 빌드 명령, 코딩 규칙, 금지 규칙 같은 기준을 정리한다.
- 세션 시작 시 전체 내용이 컨텍스트에 로드된다.
- 약 500줄 이하로 유지하는 편이 좋다.
- 그 이상이면 참조 자료는 Skill로 분리한다.

> 언제 쓰나: Claude가 항상 알아야 하는 규칙과 기준

## 2. Commands

사용자가 `/명령어`로 직접 호출하는 워크플로우 정의 파일이다.

- `.claude/commands/*.md` 경로에 마크다운 파일로 작성한다.
- 프론트매터로 `allowed-tools`, `model`, `argument-hint` 등을 지정할 수 있다.
- 하위 디렉토리로 네임스페이스를 구성할 수 있다.
- 예: `commands/ci/build.md`

> 언제 쓰나: 사용자가 직접 트리거하는 반복 워크플로우

## 3. Skills

재사용 가능한 지식과 지침을 담는 마크다운 파일이다. Claude가 문맥에 따라 자동으로 로드하거나, 사용자가 직접 호출할 수 있다.

- 세션 시작 시에는 설명(description)만 로드된다.
- 실제 호출 시 전체 내용이 로드된다.
- Subagent에 연결할 때는 시작 시 전체 내용이 미리 로드된다.
- 디렉토리 구조는 `skills/skill-name/SKILL.md` 형태다.

> 언제 쓰나: 반복 작업, 참조 문서, 때때로 필요한 지침

## 4. MCP

Claude를 외부 서비스와 도구에 연결하는 프로토콜이다.

- 데이터베이스 쿼리, Slack 게시, 브라우저 제어처럼 외부 시스템 접근이 필요할 때 쓴다.
- 세션 시작 시 연결된 서버의 도구 정의 전체가 로드된다.
- 도구 검색 기능으로 컨텍스트 사용량을 최대 10% 수준으로 제한하고, 나머지는 필요할 때 로드한다.
- Skill과 함께 쓰면 효과적이다.
- MCP는 연결을 맡고, Skill은 사용 방법을 맡는다.

> 언제 쓰나: Claude가 외부 시스템에 직접 접근해야 할 때

## 5. Subagents

격리된 컨텍스트에서 단일 작업을 수행하고 결과만 반환하는 독립 워커다.

- 주 대화와 별도의 컨텍스트 창을 사용한다.
- 메인 세션이 중간 과정으로 오염되지 않는다.
- 많은 파일 탐색이나 넓은 코드베이스 검색에 적합하다.
- Skill의 `context: fork` 설정 또는 Custom Agent 정의로 실행한다.
- 대화 기록이나 호출된 Skill을 자동 상속하지 않는다.
- 필요한 정보는 명시적으로 전달해야 한다.
- Subagents는 실행 방식이다.
- Custom Agent는 역할이 미리 정의된 Subagent다.
- Agent teams는 여러 Subagent가 협력하는 패턴이다.

> 언제 쓰나: 단일 독립 작업을 메인 컨텍스트 밖에서 처리할 때

## 6. Agent teams

독립 세션 여러 개가 공유 작업과 피어 투 피어 메시징으로 협력하는 다중 에이전트 패턴이다.

- 각 에이전트는 독립 컨텍스트에서 실행된다.
- 결과와 메시지를 서로 주고받는다.
- Subagent와 다르다.
- Subagent는 메인이 지시하고 결과를 받는 단방향 구조다.
- Agent teams는 에이전트끼리 서로 소통하며 협업하는 구조다.
- Custom Agents로 팀 멤버 역할을 정의하고, Agent teams 패턴으로 조율하는 조합이 일반적이다.
- 예: 프론트엔드 리뷰어, 백엔드 리뷰어, 보안 검토기가 병렬로 분석하고 결과를 합쳐 최종 리포트를 만든다.

> 언제 쓰나: 서로 다른 역할이 병렬로 협력해야 할 때

## 7. Custom Agents

`agents/*.md`에 정의하는 역할 특화 에이전트다.

- Subagent와 같은 구조를 가지며, 전문가 역할을 미리 설정해둔 형태다.
- `agents/*.md` 경로에 마크다운 파일로 작성한다.
- `name`, `description`, `tools`, `model` 등을 지정한다.
- 코드 리뷰어, 테스트 생성기, 보안 검토기처럼 반복적으로 쓰는 전문 역할을 정의한다.
- Custom Agent는 정의 레이어다.
- 이렇게 정의한 에이전트는 Subagent로 단독 실행할 수 있다.
- Agent teams의 팀 멤버로도 참여할 수 있다.
- 플러그인 내 `agents/` 디렉토리에도 배치해 공유하고 배포할 수 있다.

> 언제 쓰나: 같은 역할을 반복해서 쓰거나, 팀 멤버를 미리 정의해둘 때

## 8. Hooks

특정 이벤트에서 실행되는 결정론적 셸 스크립트다.

- LLM을 거치지 않고 외부에서 실행된다.
- 예측 가능하고 안정적이다.
- 도구 실행, 세션 시작과 종료, 프롬프트 제출 같은 라이프사이클 이벤트에 연결한다.
- 컨텍스트 비용은 0이다.
- 단, 출력을 대화에 추가하면 예외가 생길 수 있다.
- 예: 파일 편집 후 ESLint 자동 실행, 커밋 전 테스트 실행

> 언제 쓰나: Claude 판단 없이 항상 일어나야 하는 자동화 작업

## 9. Settings 계층

Claude Code의 동작과 권한을 제어하는 설정 체계다.

- 4단계 우선순위로 적용된다.
- `Managed` > `User` > `Project` > `Local`
- `User`: `~/.claude/settings.json`
- `Project`: `.claude/settings.json`
- `Local`: `.claude/settings.local.json`
- `allow`, `ask`, `deny`로 도구별 권한을 제어한다.
- 샌드박스, 네트워크 접근 제한, 웹 검색 차단 같은 보안 설정을 포함한다.
- Managed 설정은 사용자가 재정의할 수 없다.
- 조직 단위 정책 적용에 사용한다.

> 언제 쓰나: Claude가 무엇을 할 수 있는지 경계를 정의할 때

## 10. Plugins

Skill, Hook, Subagent, MCP 서버를 하나의 설치 단위로 묶은 패키지다.

- `/플러그인명:스킬명` 형태로 네임스페이스가 붙는다.
- 여러 플러그인을 충돌 없이 동시에 사용할 수 있다.
- 설치하면 포함된 Skill, Hook, MCP 서버가 한 번에 적용된다.
- 현재 세션에서 활성화된 플러그인은 `/plugins`로 확인할 수 있다.
- Notion, Figma, Playwright, Context7 같은 플러그인이 대표적이다.

> 언제 쓰나: 외부 서비스 연동이나 반복 워크플로우가 미리 패키징된 형태로 필요할 때

## 개념 비교 요약

| 개념 | 로드 시점 | 컨텍스트 비용 | 주요 용도 |
| --- | --- | --- | --- |
| `CLAUDE.md` | 세션 시작, 항상 | 높음 | 항상 적용할 규칙 |
| Commands | 사용자 호출 시 | 낮음 | 사용자 주도 워크플로우 |
| Skills | 호출 시, 설명만 항상 로드 | 낮음 | 참조 문서, 자동 로드 지식 |
| MCP | 세션 시작 | 중간 | 외부 서비스 연결 |
| Subagents | 온디맨드 | 격리됨 | 단일 독립 작업 실행 |
| Agent teams | 온디맨드 | 격리됨 | 다중 에이전트 협력 |
| Custom Agents | 온디맨드 | 격리됨 | 역할 정의와 재사용 |
| Hooks | 이벤트 발생 시 | 0 | 결정론적 자동화 |
| Settings | 세션 시작, 항상 | 0 | 권한과 보안 경계 설정 |
| Plugins | 설치 후 항상 | 포함 기능에 따라 다름 | Skill, Hook, MCP 묶음 활용 |
