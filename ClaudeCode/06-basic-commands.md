# 04. [초급]기본명령어 이해

자주 쓰는 내장 슬래시 명령어 모음이다.

전체 명령어는 [공식 문서](https://code.claude.com/docs/ko/commands)에서 확인할 수 있다.

## 입력 단축키

슬래시 명령어가 아닌 키 조작으로 동작하는 기능이다.

| 단축키 | 설명 | 용도 |
|---|---|---|
| `ESC` 1회 | 실행 중인 응답 중단 | 응답을 멈추고 싶을 때 |
| `ESC` 2회 | 프롬프트 전체 삭제 (빈 상태면 rewind와 동일) | 처음부터 다시 쓸 때 |
| `\` + Enter | 줄바꿈하여 계속 입력 | 여러 줄 프롬프트를 작성할 때 |
| `Ctrl+R` | 프롬프트 히스토리 검색 | 이전에 입력한 프롬프트를 다시 쓸 때 |
| `?` | 현재 환경의 단축키 목록 표시 | 단축키가 헷갈릴 때 |

## 세션 관리

| 명령어 | 설명 | 용도 |
|---|---|---|
| `/clear` | 대화 기록 삭제 | 새 주제 시작 또는 컨텍스트가 너무 길어졌을 때 |
| `/compact` | 대화를 요약해 컨텍스트 축소 | 토큰이 많이 쌓였지만 흐름은 이어가고 싶을 때 |
| `/recap` | 현재 세션 한 줄 요약 생성 (자리 비운 후 자동 표시되기도 함) | 흐름을 잊었을 때 빠르게 파악 |
| `/branch` | 현재 대화를 복사해 분기 생성 | 다른 접근법을 시도할 때 |
| `/rewind` | 현재 세션 내 이전 턴으로 되돌림 (대화·파일 선택 복원) | 잘못된 방향을 되돌릴 때. `ESC` 2회와 동일 |
| `/resume` | 종료된 세션 다시 불러오기 | 대화를 끊었다가 다시 이어갈 때, 또는 branch 후 원래 대화로 복귀 |
| `/exit` | Claude Code 종료 | |

## 설정·환경

| 명령어 | 설명 | 용도 |
|---|---|---|
| `/model` | 사용 모델 변경 | 작업 성격에 따라 속도·성능 조절 |
| `/model opusplan` | 계획·추론은 Opus, 구현은 Sonnet으로 자동 전환 | 복잡한 작업에서 추론 품질과 실행 속도를 함께 챙길 때 |
| `/fast` | Fast 모드 ON/OFF | 같은 모델에서 더 빠른 출력이 필요할 때 |
| `/btw` | 메인 컨텍스트에 추가하지 않고 사이드 질문 | 흐름을 끊지 않고 빠르게 확인할 때 |

## 시나리오별 활용

**대화가 너무 길어졌을 때** — 토큰 과다, 응답 지연, 흐름 이탈
- 흐름을 이어가려면: `/compact`로 요약 후 계속
- 주제를 바꾸려면: `/clear`로 초기화
- 무엇이 컨텍스트를 차지하는지 먼저 보려면: `/context`로 현황 확인 후 판단

**복잡한 구조 설계 전** — 추론 품질과 실행 속도를 함께 챙겨야 할 때
- `/model opusplan`으로 전환 → 계획·추론은 Opus, 코드 생성은 Sonnet이 자동으로 담당

**다른 접근법을 시도해보고 싶을 때** — 현재 맥락을 잃지 않고 실험
- `/branch [이름]`으로 현재 대화를 복사해 분기 → 다른 방향 시도
- 원래 대화로 돌아오려면: `/resume`으로 복귀

**구현 직후 코드 정리** — 커밋 전 품질 점검
- 변경 범위 확인: `/diff`
- 코드 리뷰: `/review` (수정 없음), 보안 점검 포함 시 `/security-review` 추가
- 재사용성·효율성까지 정리: `/simplify` 실행

**Claude가 엉뚱한 방향으로 응답했을 때** — 방향 수정
- `ESC` 2회로 직전 턴 rewind → 프롬프트 재작성

**흐름을 끊지 않고 빠르게 확인할 때** — 메인 작업과 무관한 사이드 질문
- `/btw`로 시작 → 답변은 받되 메인 컨텍스트에는 추가되지 않음

**내 사용 패턴을 파악할 때** — 주로 어떤 작업에 쓰는지 점검
- `/insights`로 세션 분석 리포트 생성
